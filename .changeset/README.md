# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works with multi-package repos, or single-package repos to help you version and publish your code. You can find the full documentation for it [in our repository](https://github.com/changesets/changesets).

We have a quick list of common questions to get you started engaging with this project in [our documentation](https://github.com/changesets/changesets/blob/main/docs/common-questions.md).

## How it works in this repo

Every PR that changes published code under `packages/*` must include a changeset:

```bash
npx changeset
```

The interactive prompt walks you through picking the affected package(s), the semver bump (patch / minor / major), and writing a one-line summary. Commit the resulting `.changeset/<random>.md` file alongside your code change.

When the PR merges to `main`, the `release.yml` workflow opens (or updates) a "Version Packages" PR that:

- bumps the affected `packages/*/package.json` versions
- prepends the changeset summary to the per-package `CHANGELOG.md`
- consumes the `.changeset/*.md` file

Merging that PR triggers `changeset publish` → `npm publish` for each affected package.

Per-package independent versioning: `@sawala/formulir-react@0.4.x` and `@sawala/kontena-client@0.1.x` can coexist with unrelated version timelines.
